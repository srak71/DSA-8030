Exploratory Data Analysis

Exploratory Data Analysis refers to the critical process of performing initial investigations on data so as to discover patterns,to spot anomalies,to test hypothesis and to check assumptions with the help of summary statistics and graphical representations.

There are many types of Exploratory Data Analysis that include graphical representations which we have already begun to explore. In addition to these graphical representations there are non-graphical techniques that describe the data and help uncover patterns and other features.

We will begin to to discuss these non-graphical techniques first with univariable teachiniques then multivariable techniques.

Univariate Techniques

Numerical Variables

You may recall that for any single numerical variable a researcher should report on the center, shape, and spread (variability) of the variable distribution. The researcher should also mention any unusual features in the distribution. We have explored the shape portion of this and will now look at statistical measures for center and variability.

Using Base R

Within base R there are singular functions that will calculate mean, median, standard deviation, etc. for a single or multiple univariate calculations.

These functions include but are not limited to mean(), median(), sd(), etc.

The functions in base R were not created to use piping. Therefore, we need to use other techniques to obtain the columns we are looking to analyze. For example, if we want to explore the loan_amount from the loans dataframe then we can indicate a $ to represent that we are exploring the loans dataset, the loan_amount variable.

# Mean Loan Amount

mean(loans$loan_amount)
## [1] 16361.92

This code of the mean() function (and the others) should work fine as long as there are no missing values in your variable. If you do have missing values (as in debt_to_income) you will see this…

# Mean Debt to Income 

mean(loans$debt_to_income)
## [1] NA

This is because if you include a missing value in the calculations the returned statistics is missing. To combat this you can use an argument na.rm. When this argument is set to TRUE the NAs are removed from the calculation of the statistic.

# Mean Debt to Income 

mean(loans$debt_to_income, na.rm=TRUE)
## [1] 19.30819

Using summarize function from tidyverse

The summarise function in the dplyr package of tidyverse allows you to collapse rows down to summary statistics.

# Mean Loan Amount

loans |>
  summarise(mean_loan_Amount = mean(loan_amount))

Let’s break down this code:

  • The first argument of the summarize function (as is typical with tidyverse functions) is the dataset. This argument is piped into the function.

  • The second argument of the summarize function is an equation. The left of the equation is a label you choose for the summary statistic. The right of the equation is the function you wish to calculate. Recall mean function is from base R and is the same function we used above. Therefore, if we have NA values you will need to accomodate for those NAs.

# Mean of debt to income
# Not accounting for NAs

loans |>
  summarise(mean_dti = mean(debt_to_income))
# Accounting for NAs

loans |>
  summarise(mean_dti = mean(debt_to_income, na.rm=TRUE))

You may be asking yourself why would I use the summarise function above just using the mean function. It seems like a LOT more typing. There are a few advantages. One, the summarise function can accomodate piping therefore may be easy to incorporate with other functions. Two, you can combine functions within the summarise function to create your own customized summary.

loans |>
  summarise(mean_loan_Amount = mean(loan_amount), 
            sd_loan_Amount = sd(loan_amount), 
            median_IR = median(interest_rate), 
            count = n())

Each addtional argument in the summarise function is a new label and statistic calculation.

Multivariate Techniques

We can also find summary statistics for numerical variables and categorical variables by groups (a different categorical variable).

Numerical Variables

To summarize numeric variables based on a categorical variable we can use the group_by function in tidyverse. In the following code we are calculating summary statistics for loan_amount for each homeownership category.

loans |>
  group_by(homeownership) |>
  summarise(mean_loanAmt = mean(loan_amount), 
            sd_loanAmt = sd(loan_amount), 
            min_loanAmt = min(loan_amount),
            max_loanAmt = max(loan_amount),
            n = n()
            )

You can use group_by in other pipable functions as well…

loans |>
  group_by(homeownership) |>
  rstatix::get_summary_stats(loan_amount, type = "mean_sd")
